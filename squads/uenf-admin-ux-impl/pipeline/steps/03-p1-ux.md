# Step 3 — Melhorias P1 (UX/UI das Páginas Admin)

## Agente
Diana Design 🎨

## Tarefas

### P1-05: Agrupar extensões por categoria
**Arquivo:** `functions.php` — função `cct_extensions_page_callback` (~linha 1439–1466)
**Problema:** 13 extensões em lista plana, sem contexto visual de relação entre elas.
**Solução:** Usar o campo `category` de cada extensão para agrupar visualmente.
**Categorias e suas extensões:**
- **Design:** Gerenciador de Cores, Sistema de Ícones, Modo Escuro/Claro, Sistema de Sombras, Sistema de Gradientes, Design Tokens
- **Layout:** Responsive Breakpoints
- **Tipografia:** Tipografia Avançada, Combinações de Fontes
- **Efeitos:** Sistema de Animações
- **Interface:** Sistema de Busca Personalizado
- **Conteúdo:** Biblioteca de Padrões

**Critério de Done:** Tabela exibe cabeçalhos de grupo entre as extensões.

---

### P1-07: Padronizar visual da página Reset
**Arquivo:** `functions.php` — função `cct_reset_page_callback` (~linha 2550–2644)
**Problema:** Página Reset usa `.wrap` genérico do WordPress. Extensões e Docs têm header gradiente + cards.
**Solução:** Adicionar header com gradiente azul-roxo igual às outras páginas + cards para cada tipo de reset.
**Referência visual:** Ver o header da função `cct_extensions_page_callback` e replicar o padrão HTML/CSS.

**Critério de Done:** Página Reset tem visual consistente com as demais (header gradiente, cards brancos).

---

### P1-06: Corrigir parser Markdown
**Arquivo:** `functions.php` — função de parse Markdown (~linhas 827–837)
**Problema A:** Flag `/s` (dotall) na regex de `<ul>` cria listas aninhadas incorretas quando há múltiplas listas.
**Problema B:** Headings não recebem `id` automático — links da navegação interna não funcionam.
**Solução A:** Remover flag `/s` da regex de listas.
**Solução B:** Na substituição de h1–h4, adicionar `id="{slug}"` usando `sanitize_title()`.

**Critério de Done:** Listas renderizam corretamente. Links do índice navegam para as seções.

---

### P1-08: Adicionar link Documentação no Acesso Rápido
**Arquivo:** `functions.php` — função `cct_admin_page_callback` (~linhas 774–778)
**Problema:** Card "Acesso Rápido" tem apenas 2 botões (Extensões, Personalizar). A Documentação existe mas não aparece ali.
**Solução:** Adicionar terceiro botão "Documentação" apontando para `admin.php?page=tema-uenf-docs-design`.

**Critério de Done:** Página principal do painel tem 3 botões no Acesso Rápido.

---

### P1-04: Remover painel vazio de extensões no Customizer
**Arquivo:** `inc/extensions/class-extension-manager.php`
**Problema:** Método `add_customizer_controls()` registra painéis `cct_theme_uenf` e `cct_active_extensions` sem nenhum controle visível (todo o código de controles está comentado). Usuário vê painel vazio sem utilidade.
**Solução:** No método `add_customizer_controls()`, remover ou comentar o registro dos painéis `cct_theme_uenf` e `cct_active_extensions` sem afetar outras funcionalidades.

**Critério de Done:** Customizer não exibe o painel "Tema UENF" vazio.
